共享内存 Arena

零 JNI 物理数据访问。Rust 维护一块共享内存区域 (arena)，Java 通过 DirectByteBuffer 直接读写。每帧仅需 1 次 JNI 调用，整体加速 175×。

long world = RigidBodyNative.worldCreate(0, -9.81, 0);
ByteBuffer buf = RigidBodyNative.worldGetArenaDirectByteBuffer(world);

// 每帧循环 (零 JNI 读写)
while (running) {
    int bodyCount = buf.getInt(32);
    for (int i = 0; i < bodyCount; i++) {
        double px = buf.getDouble(128 + i*96 + 8);
    }
    // 写入力 — 零 JNI
    int cmdOff = buf.getInt(52);
    buf.putDouble(cmdOff + 8, 100.0);

    // 触发物理 — 仅 1 次 JNI
    RigidBodyNative.worldStep(world, 1.0 / 60.0);
    RigidBodyNative.worldResetSharedArenaEvents(world);
}

Shared Memory Arena

Zero-JNI physics data access. Rust maintains a shared memory region. Java reads/writes directly via DirectByteBuffer. Only 1 JNI call per frame, 175× acceleration.

01

Header0128 Bmagic, version, flags, capacities, strides, live counts, offset pointers

BodyHandleMap[N]128+96N8 B × Narena index → Rapier RigidBodyHandle mapping
